1. SortedIntList-ийн хэрэгжилтийн нарийн зүйлс дээр аль нь илүү хамааралтай вэ: composition эсвэл inheritance?

-Inheritance нь SortedIntList-ийн дотоод хэрэгжүүлэлтээс маш их хамааралтай. Учир нь эцэг классын аргууд бие биенээ дуудаж буй (self-invocation) логикийг заавал мэдэх шаардлагатай болдог. Харин Composition нь объектыг "хар хайрцаг" мэт ашигладаг тул дотоод хэрэгжилтээс хамаарал багатай, илүү бие даасан байдаг.

2. Хэрэв SortedIntList-ийн add аргыг ихээхэн өөрчилж эсвэл үйлдэл нь өөрчлөгдөх бол аль хэрэгжилт илүү их алдаа гаргах магадлалтай вэ?

- Inheritance илүү их алдаа гаргах эрсдэлтэй (Fragile Base Class problem). Хэрэв эцэг классын add эсвэл addAll-ийн харилцан хамаарал өөрчлөгдвөл, түүнийг өвлөж авсан классын тоолуур (instrumentation) шууд буруу тоолж эхэлнэ.

3. Inheritance ашиглахад үүссэн асуудлыг composition ашиглан ямар асуудлыг шийдвэрлэж чадна вэ?

- Composition нь классуудын хоорондын "хэт нягт холбоо" (tight coupling)-ыг салгаж өгдөг. Эцэг классын аргууд дотроо бие биенээ дуудах нь Composition-д нөлөөлдөггүй тул "Double Counting" (хоёр дахин тоолох) зэрэг логик алдаанаас бүрэн сэргийлж, кодын хяналтыг илүү найдвартай болгодог.

4. Өгөгдсөн хэрэгжилтийн үндсэн дээр, аль үед inheritance-г, аль үед composition-г ашиглах нь илүү зохистой вэ?

- Inheritance: Классуудын хооронд маш тодорхой "Is-A" (нь мөн) харилцаа байгаа бөгөөд эцэг классын бүх шинж чанарыг өөрчлөлтгүй ашиглах үед.

- Composition: Классуудын хооронд "Has-A" (той) харилцаа байгаа үед буюу өөр классын үйлдлийг зөвхөн ашиглах (instrumentation), өргөтгөх шаардлагатай үед.
